利用規約

最終更新日: 2025年7月12日
サービス概要: 「松蔭（showIN）」は、Amazon Alexa を通じて音声でメモを管理できる無料のスキルサービスです。

1. 利用の前提条件

本サービスをご利用いただくには、以下の条件を満たす必要があります：

  • 対応デバイス: Amazon Echo シリーズまたはAlexa アプリがインストールされたデバイス
  • ネットワーク: 安定したインターネット接続環境
  • アカウント: 有効なAmazon アカウント
  • 言語設定: Alexa の言語設定が日本語であること
  • 年齢制限: 13歳以上（未成年者は保護者の同意が必要）

2. サービス内容

本サービスでは以下の機能を提供します：

  • メモ追加: 音声コマンドによるメモの作成・保存
  • メモ読み上げ: 保存されたメモの音声での再生
  • メモ削除: 指定したメモの削除
  • ヘルプ機能: 使用方法の音声ガイダンス
無料サービス: 本サービスは完全に無料で提供されます。利用者に料金を請求することはありません。

3. 利用者の責任

サービスをご利用の際は、以下にご注意ください：

  • 適切な使用: 本来の目的（メモ管理）以外での使用は控えてください
  • 個人情報の注意: 機密情報や個人情報をメモに含める際は十分ご注意ください
  • 音声環境: 明瞭な音声コマンドの実行にご協力ください
  • デバイス管理: Alexa デバイスの適切な管理・設定

4. 禁止事項

以下の行為を禁止します：

  • 不正利用: システムの脆弱性を悪用した攻撃・侵入
  • 過度な負荷: サーバーに異常な負荷をかける行為
  • 違法コンテンツ: 法令に違反する内容のメモ作成
  • 第三者の権利侵害: 他者の著作権・プライバシー等の侵害
  • 商用利用: 営利目的での本サービスの使用
  • 逆アセンブル: システム構成の解析・複製の試み
違反時の対応: 禁止事項に該当する行為が確認された場合、事前通知なくサービス利用を停止する場合があります。

5. サービスの制限

以下の技術的・運用的制限があります：

  • 可用性: 24時間365日の稼働を保証するものではありません
  • 音声認識精度: 環境により認識精度が変動する場合があります
  • メモ内容: 極端に長いメモは適切に処理されない場合があります
  • 同時接続: 同一アカウントからの同時アクセスに制限がある場合があります
  • データ量: 大量のメモ保存時はパフォーマンスが低下する場合があります

6. 免責事項

以下について一切の責任を負いません：

  • データ損失: システム障害・メンテナンス等によるメモデータの消失
  • サービス中断: 予期しないサービス停止による影響
  • 音声認識エラー: 誤認識によるメモ内容の相違
  • 第三者サービス: Amazon Alexa プラットフォームの問題による影響
  • 機会損失: サービス利用または利用不能による間接的損害
  • ネットワーク障害: 通信環境の問題による利用支障

7. サービスの変更・終了

  • 機能変更: 事前の通知により機能の追加・変更・削除を行う場合があります
  • サービス終了: 30日前の事前通知により、サービス全体を終了する場合があります
  • データ移行: サービス終了時のデータ移行手段は提供されません
  • 緊急対応: セキュリティ上の理由により、緊急でサービスを停止する場合があります

8. 知的財産権

  • サービス著作権: 本サービスのシステム・ドキュメント等の著作権は開発者に帰属
  • メモ著作権: 利用者が作成したメモの著作権は利用者に帰属
  • 商標権: 「Alexa」「Amazon」等の商標は各権利者に帰属
  • ライセンス: オープンソースライセンスに基づく部分はそれぞれのライセンスに従います

9. 規約の変更

本規約は以下の方法で変更される場合があります：

  • 通知方法: 本ページでの告知・Alexa アプリ内通知
  • 効力発生: 変更通知から30日後に新規約が効力を発生
  • 継続利用: 変更後もサービスを利用された場合、新規約に同意したものとみなします

10. 準拠法・管轄

  • 準拠法: 日本法に準拠します
  • 管轄裁判所: 東京地方裁判所を専属管轄裁判所とします
  • 紛争解決: まずは誠実な協議による解決を試みます

11. お問い合わせ

本規約に関するご質問は、以下の方法でお問い合わせください：

  • GitHub Issues: プロジェクトページ
  • Alexa アプリ: スキルレビュー機能
開発方針: 本サービスは ideanotes の「スモールスタート原則」に基づき、利用者の声を重視した継続的改善を行います。
